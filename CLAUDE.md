## Architecture

This is an interactive CLI tool for managing Bitbucket and GitHub repositories. Entry point is `main.py`, which presents a numbered menu and dispatches to class-based handlers.

**Data sources:**
- `accounts.csv` — Bitbucket account credentials (gitignored; columns: `email`, `workspace`, `project_key`, `username`, `app_password`, `is_private`)
- `.env` — GitHub credentials (`GITHUB_TOKEN`, `GITHUB_USERNAME`)
- `{email}_repos.csv` — Cached repo lists per Bitbucket account (generated by the app)
- `github_repos.csv` — Cached GitHub repo list (generated by the app)

**Class structure:**

Each menu option instantiates a dedicated orchestrator class from `classes/`. These all follow the pattern: `__init__` calls `self.start()` which drives the full interactive flow. The orchestrators delegate to two shared backends:

- `classes/Bitbucket.py` — Shared Bitbucket backend: account resolution, CSV read/write, repo selection via fzf, git operations (clone, mirror, push). Always fetches credentials from `accounts.csv` via `AccountsCsv` before calling the API.
- `classes/GithubClass.py` — Shared GitHub backend: repo export to CSV, clone, create, delete, mirror push. Reads credentials from `.env`. Note: GitHub username `seriiserii825` is hardcoded in several methods alongside `.env`-sourced values.

**Supporting layers:**

- `classes/AccountsCsv.py` — Reads `accounts.csv`, provides `choose_account_by_email()` via `simple-term-menu`
- `classes/BitbucketApi.py` — Raw HTTP calls to Bitbucket REST API v2.0 using `HTTPBasicAuth`
- `classes/Browser.py` — Opens browser URLs with `webbrowser.open()` and waits for user confirmation (y/n prompts). Used for repo creation and group permission steps that have no API equivalent.
- `classes/Select.py` — Thin wrapper around fzf (multi-select) and questionary; consumed by `utils.selectMultiple`
- `modules/git_mirror.py` — Legacy standalone functions for `git clone --mirror` / `git push --mirror`; largely superseded by the class-based orchestrators
- `utils.py` — UI helpers: `selectOne` (simple-term-menu), `selectMultiple` (fzf), `pretty_print`, `pretty_table` (rich)
- `views.py` — Thin wrapper functions around class methods; mostly unused since `main.py` instantiates orchestrator classes directly
- `my_types/` — `NamedTuple` definitions: `AccountType`, `RepoType`
- `execeptions/` — Custom exceptions: `AccountException`, `BitbucketException`, `BitbucketApiException`, `GithubException` (directory name has a typo — keep as-is)

**Typical data flow:**
`AccountsCsv` → `AccountType` → `BitbucketApi(username, app_password)` → API response → write to `{email}_repos.csv` → fzf selection for subsequent operations.

**Mirror workflow** (options 2 and 13): clones with `git clone --mirror`, `cd`s into the `.git` directory, opens browser for manual repo creation and group permission steps, then runs `git push --mirror`. For Bitbucket-to-GitHub (option 13), also creates the GitHub repo via API before pushing.
